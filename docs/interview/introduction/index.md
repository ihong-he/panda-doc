## 2、自我介绍（版本二）

面试官好（你们好），我叫XXX，本科学历，网络工程专业。拥有近五年前端开发经验，上家公司在博欣科技担任软件开发工程师的岗位，主要负责中大型项目的技术选型、架构设计与核心模块开发工作。

在过去两年中，我主导了博欣信息管理系统的前端架构设计与核心功能落地，覆盖 PC 端与移动端，支撑企业进销存、物料管理、生产制造等多个业务模块。同时，我独立负责了试剂管理系统桌面端应用的从零搭建，基于 Electron 与 Vue3 实现了试剂全生命周期管理。此外，我还主导了博欣项目管理系统的技术选型与开发，基于 React 生态完成了核心功能的实现。

然后我介绍一下最近做的博欣信息管理系统，这是一个面向制造研发类企业的全流程企业管理系统，主要解决企业从客户下单到产品交付的整个流程的数字化管理问题。

我主要负责前端架构设计和核心功能开发，具体做了这么几块：

第一块是核心业务模块。我做了进销存、BOM物料管理、生产制造这些模块。其中BOM系统是比较复杂的，要处理产品物料清单的多层级展开关系，支持很深的树形结构，这是制造企业最核心的功能。

第二块是权限管理系统。这是企业系统最关键的部分。我设计了一套基于RBAC的细粒度权限控制，包括三个层级：菜单权限——不同角色看到的菜单不一样；按钮权限——比如普通员工只有查看按钮，管理员有删除权限；字段权限——不同岗位的人看到的表格列也不同。

第三块是性能优化。开发过程中遇到比较严重的性能问题，BOM树的数据量可能达到几万条，用传统表格渲染浏览器直接卡死。后来我用了虚拟滚动技术，只渲染屏幕上可见的行，这样即使10万条数据也能流畅滚动，把加载时间从8秒优化到了1.5秒。

第四块是大文件处理。制造企业的工程图纸文件特别大，从100MB到2GB都有。我实现了分片上传加上断点续传再加秒传这三个功能：分片上传是把大文件切成小片并行上传；断点续传是支持断网后接着传，不用从头开始；秒传是如果服务器已经有相同文件就直接返回链接，不用再传一遍。这套方案把上传成功率提升到了98%。图纸预览这边我集成了kkFileView，在服务端把文件转成HTML，前端用iframe嵌入，这样浏览器不用加载大文件，预览体验很流畅。

第五块是移动端开发。用UniApp开发了移动端，一套代码同时适配H5和小程序，让员工在手机上也能处理审批、查库存这些工作。

技术栈方面，PC端用的是Vue3 + Vite + Pinia + Element Plus + TypeScript，移动端是UniApp + Vue3，可视化用了ECharts和LogicFlow流程图。
