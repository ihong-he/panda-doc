## 一、ES6（必考重点）

### 1、ES6核心特性（面试高频前5名）

::: danger 🔥 重要提醒
现在企业面试基本都会问ES6，这5个是重中之重：
:::

**必考1：let/const** 📝
- 解决var的痛点：变量提升、重复声明、函数作用域
- let有块级作用域，不存在变量提升
- const声明常量，但对象数组内容可改（地址不能改）

**必考2：箭头函数** ➡️
- 语法更简洁：`(a, b) => a + b`
- 没有自己的this，继承外层this（这是最重要的考点！）
- 不能当构造函数，没有arguments对象

**必考3：解构赋值** 🎯
- 数组解构：`const [a, b] = [1, 2]`
- 对象解构：`const {name, age} = obj`
- 实际开发中特别常用，尤其是React项目

**必考4：模板字符串** 💬
- 用反引号：`` `Hello ${name}` ``
- 支持多行字符串，再也不用拼接了

**必考5：Promise** 🔄
- 解决回调地狱，异步编程的利器
- 三种状态：pending、fulfilled、rejected
- then/catch/finally用法

::: info 📚 其他特性（了解即可）
- 模块化：import/export
- 展开运算符：`...args`
- 新的数据类型：Map、Set、Symbol
:::

### 2、let、const、var区别（经典面试题）

::: warning 🚨 经典面试题
这个是面试必考题！一定要记住这3个关键区别：
:::

**作用域不同** 🌐
- var：函数作用域，在函数内部都有效
- let/const：块级作用域，只在`{}`内有效

**变量提升** ⬆️
- var：会提升，声明前可用（值是undefined）
- let/const：不会提升，声明前报错（暂时性死区）

**重复声明** 🔄
- var：可以重复声明，后声明的会覆盖前面的
- let/const：不能重复声明，会报错

**const的特殊性** 🔒
- 基本类型：值不能改
- 引用类型：地址不能改，但内容可以改！
```javascript
const arr = [1, 2, 3];
arr.push(4); // ✅ 可以
arr = [4, 5]; // ❌ 报错，改变了地址
```

::: tip 💡 面试技巧
记住"用const优先，需要重新赋值时用let，基本不用var"这个原则！
:::

### 3、箭头函数 VS 普通函数（超级重点）

::: danger 🔥 超级重点
这个区别每年面试都会考，特别是this的指向问题！
:::

**核心区别1：this指向** 🎯
- 箭头函数：没有自己的this，继承外层的this
- 普通函数：this由调用方式决定（谁调用就指向谁）

**核心区别2：语法简洁** ✨
- 箭头函数一行搞定：`const add = (a, b) => a + b`
- 普通函数需要写function和return

**核心区别3：构造函数** 🏗️
- 箭头函数：不能当构造函数，不能用new
- 普通函数：可以当构造函数

**核心区别4：arguments** 📋
- 箭头函数：没有arguments，可以用剩余参数代替
- 普通函数：有arguments对象

::: tip 🎯 面试经典场景
```javascript
const obj = {
  name: '张三',
  sayName: function() {
    console.log(this.name); // 张三
    setTimeout(() => {
      console.log(this.name); // 张三（箭头函数继承外层this）
    }, 100);
  }
};
```
:::

::: info 💡 记忆口诀
"箭头函数无this，外层this就是它；
普通函数this活，谁调用指向家"
:::



### 4、Promise 和 async/await（异步编程必考）

::: danger 🔥 必考内容
异步编程是现代前端开发的核心，Promise + async/await是必备技能！
:::

**Promise基础** 🔄
- 三种状态：pending（等待）→ fulfilled（成功）或 rejected（失败）
- 状态一旦确定就不能再改
- 链式调用：then().catch().finally()

**async/await优势** ⚡
- Promise的语法糖，让异步代码看起来像同步
- 更容易阅读和维护
- 错误处理用try/catch，比catch更直观

**🎯 面试重点**
1. **并发执行**：Promise.all() vs Promise.race()
   - all：全部成功才成功，一个失败就失败
   - race：谁快就返回谁的结果

2. **错误处理**
   - Promise用catch捕获错误
   - async/await用try/catch

3. **执行顺序**
   - 同步代码 > 微任务(Promise) > 宏任务(setTimeout)

::: tip 💡 实际应用
现在的项目基本都是Promise + async/await混合使用，这个一定要会！
:::

### 5、ES6模块与CommonJS的区别？

::: danger 🔥 必考区别题
这是模块化面试的高频考点，理解二者的差异很重要！
:::

**核心区别** 🔄

**1. 语法不同** 📝
- **ES6模块**：`import/export`
  ```javascript
  import { name } from './module';
  export default function() {}
  ```
- **CommonJS**：`require/module.exports`
  ```javascript
  const { name } = require('./module');
  module.exports = function() {}
  ```

**2. 加载时机** ⏰
- **ES6模块**：编译时确定依赖关系（静态）
- **CommonJS**：运行时加载（动态）

**3. 值的引用** 🔗
- **ES6模块**：值的引用，原始值变化会反映
- **CommonJS**：值的拷贝，获取时就是当时的值


**🎯 面试重点**

**静态 vs 动态**
```javascript
// ES6静态 - 可以在编译时优化
import { utils } from './utils'; // ✅ 编译时确定

// CommonJS动态 - 运行时才能确定
const utils = require('./utils'); // ❓ 运行时确定
```

**实际应用** 💡
- **浏览器环境**：主要使用ES6模块
- **Node.js环境**：传统用CommonJS，现在也支持ES6模块
- **构建工具**：webpack等工具会将ES6模块转换为CommonJS等格式

::: tip 💡 面试回答技巧
强调"ES6模块是静态的，CommonJS是动态的"这个核心差异，以及由此带来的性能优化优势。
:::




## 二、网络请求（前端必知必会）

### 1、HTTP VS HTTPS（基本概念题）

::: tip 📚 基础概念
这个现在算是送分题，但一定要答对！
:::

**核心区别：安全性** 🔒
- HTTP：明文传输，像明信片一样谁都能看
- HTTPS：加密传输，像保险箱一样安全

**其他区别** 📋
1. **端口号不同** 🔌
   - HTTP：80端口
   - HTTPS：443端口

2. **证书验证** 📜
   - HTTP：没有证书，无法验证身份
   - HTTPS：有SSL证书，验证服务器身份

3. **性能差异** ⚡
   - HTTP：稍快（无加密开销）
   - HTTPS：稍慢（需要加密解密）

::: tip 🎯 面试加分项
提到SSL/TLS握手过程，说明HTTPS是如何建立安全连接的。
:::

::: info 💡 现状
现在基本所有网站都用HTTPS，HTTP已被浏览器标记为"不安全"！
:::



### 2、GET VS POST（高频面试题）

::: warning 🚨 高频面试题
这是面试必考题，特别是面试官喜欢问区别！
:::

**核心区别1：数据位置** 📍
- GET：参数放在URL后面，所有人都能看到
- POST：参数放在请求体里，相对隐蔽

**核心区别2：数据大小** 📏
- GET：URL有长度限制（一般2KB左右）
- POST：没有限制，可以传大文件

**核心区别3：安全性** 🔐
- GET：不安全，参数会出现在浏览器历史记录
- POST：相对安全，但绝对安全需要HTTPS

**核心区别4：幂等性** 🔄
- GET：幂等，多次请求结果一样
- POST：非幂等，每次提交可能创建新数据

**核心区别5：缓存** 💾
- GET：可以被浏览器缓存
- POST：不会被缓存

::: tip 🎯 面试技巧
记住"GET查数据，POST提交数据"这个基本原则！
:::

::: info 💡 实际应用
- 搜索、分页用GET
- 登录、注册、文件上传用POST
:::


### 3、HTTP状态码（必须掌握的10个）

::: tip 📚 必备知识
面试不需要背全部状态码，记住这10个常用的就够了！
:::

**2xx成功类** ✅
- 200：一切正常，请求成功
- 201：创建成功（比如注册用户）
- 204：成功但无内容返回（比如删除操作）

**3xx重定向类** 🔄
- 301：永久搬家，浏览器会自动换新地址
- 302：临时搬家，浏览器还会用老地址

**4xx客户端错误（常见！）** ❌
- 400：请求有问题，参数不对或格式错误
- 401：没登录，需要身份验证
- 403：登录了但没权限访问
- 404：页面不存在，最常见的错误

**5xx服务器错误** 🚨
- 500：服务器内部崩溃了
- 502：网关问题，通常是服务器挂了

::: tip 🎯 记忆技巧
- 401：没带钥匙（未认证）
- 403：带了钥匙但开不了这扇门（无权限）  
- 404：房子不存在（资源不存在）
:::

::: warning 💡 面试重点
重点说401/403/404的区别，这个面试官特别喜欢问！
:::


### 4、URL组成（基础概念题）

::: info 📝 基础概念
这个比较简单，但一定要答完整！
:::

**URL的6个组成部分** 🔗

**协议** 🌐：怎么访问（http/https/ftp）

**域名** 🏠：访问谁（www.baidu.com）

**端口** 🚪：哪个门（http默认80，https默认443）

**路径** 📍：具体位置（/user/login）

**参数** 📋：额外信息（?id=1&name=test）

**锚点** 🔗：页面内定位（#section1）

::: tip 🎯 实例解析
```
https://www.example.com:8080/api/user?id=123#profile
```
- 协议：https
- 域名：www.example.com  
- 端口：8080
- 路径：/api/user
- 参数：id=123
- 锚点：#profile
:::

::: info 💡 记忆口诀
"协议域名端口，路径参数锚点"
:::

### 5、跨域解决方案（必考实战题）

::: danger 🔥 必考实战题
跨域是每个前端都会遇到的问题，面试必考！
:::

::: info ❓ 什么是跨域？
浏览器出于安全考虑，不允许不同源（协议、域名、端口不同）的页面之间互相访问。
:::

::: info 🛡️ 为什么会有跨域？
防止恶意网站窃取用户信息，保护用户安全。
:::

**三种主要解决方案** 🛠️

**1. CORS（最常用）** 🌐
- 服务器配置响应头，告诉浏览器"我允许这个域访问我"
- 关键响应头：`Access-Control-Allow-Origin: *`
- 生产环境一般用具体域名，不用`*`

**2. 代理服务器（开发常用）** 🔄
- 前端请求自己的服务器，自己的服务器再去请求真实API
- webpack.devServer.proxy配置
- 开发环境用得多，避免每次都改服务器配置

**3. JSONP（古老方法）** 📜
- 只支持GET请求
- 利用script标签不受同源策略限制的特性
- 现在基本不用，了解即可

::: tip 🎯 面试重点
- 跨域是浏览器限制，服务器之间不存在跨域
- CORS是主流方案，代理是开发便捷方案
- 简单请求和复杂请求的区别（预检请求）
:::

### 6、输入URL到页面显示全过程（经典流程题）

::: danger 🔥 经典流程题
这个题考察对整个浏览器工作原理的理解，按顺序说清楚就好！
:::

**完整流程（5步记熟）** 🔄

**第1步：DNS解析** 🔍
- 把域名（www.baidu.com）变成IP地址（220.181.38.148）
- 先查浏览器缓存 → 操作系统缓存 → 路由器缓存 → DNS服务器

**第2步：建立TCP连接** 🤝
- 三次握手建立连接
- 我要连接（SYN）→ 可以连接（SYN-ACK）→ 确认连接（ACK）

**第3步：发送HTTP请求** 📤
- 浏览器发送请求报文
- 包含请求行、请求头、请求体

**第4步：服务器处理响应** 📥
- 服务器处理请求，返回响应
- 包含状态码、响应头、响应体（HTML内容）

**第5步：页面渲染** 🎨
- 解析HTML → 构建DOM树
- 解析CSS → 构建CSSOM树  
- 合并成渲染树 → 布局 → 绘制
- 遇到JS会阻塞渲染，执行完再继续

::: tip 🎯 关键点
- DNS解析是第一步
- TCP连接是基础
- 渲染过程会涉及重排和重绘
- JS执行会阻塞页面渲染
:::

::: info 💡 加分项
可以提一下HTTP缓存、CDN、浏览器的重排重绘优化等。
:::

### 7、WebSocket理解（实时通信）

::: tip 🚀 现代技术
WebSocket是现在很火的实时通信技术！
:::

**WebSocket是什么？** 📡
一种全双工通信协议，客户端和服务器可以互相主动发消息。

**和HTTP的区别** ⚡
- HTTP：只能客户端问，服务器答（单向）
- WebSocket：客户端和服务器都能主动发消息（双向）
- HTTP：每次请求都要建立新连接
- WebSocket：一次连接，持续通信

**主要优势** ✨
1. **实时性强**：消息立即推送，不用轮询
2. **开销小**：不用每次都建立连接
3. **双向通信**：服务器也能主动推送消息

**使用场景** 🎯
- 聊天室（微信网页版）
- 在线协作（多人编辑文档）
- 实时数据（股票价格、游戏状态）
- 在线教育（直播弹幕）

**连接过程** 🔗
1. 先用HTTP发起WebSocket连接请求
2. 服务器同意后升级为WebSocket协议
3. 后续通信都用WebSocket协议

::: tip 🎯 面试重点
对比HTTP长轮询，说明WebSocket的优势在于真正的实时性。
:::
