## 四、ArkTS 基础

### 4.1 工程结构与默认代码

#### 核心目标

了解基于HarmonyOS的工程结构，读懂默认代码含义

#### 关键目录

- 代码文件目录：`entry/src/main/ets/pages`（所有业务代码都存放在这里）
- 资源文件目录：`entry/src/main/resources`（图片、文字等资源存放处）

#### 鸿蒙主开发语言

**ArkTS**（基于TypeScript进行扩充和提升），学习路径：`ArkTS语法基础 → 布局界面 → ArkTS核心交互`

### 4.2 数据类型与变量

#### 核心目标

使用变量存储不同类型的数据

#### 常见数据类型

- 字符串类型（string）：存储文字信息，如“创意橘子花瓶”
- 数字类型（number）：存储数字信息，如1、12.99
- 布尔类型（boolean）：存储状态信息，只有true（是）和false（否）两种值

#### 变量语法规则

```TypeScript
let 变量名: 数据类型 = 值
```

示例：

- `let title: string = '创意橘子花瓶'`（存储商品名称）
- `let count: number = 1`（存储商品数量）
- `let isSelect: boolean = true`（存储是否选中状态）

### 4.3 数组

#### 核心目标

一次性保存多个同类型数据

#### 数组语法规则

```TypeScript
let 数组名: 类型[] = [数据1, 数据2, 数据3, ……]
```

示例：`let titles: string[] = ['创意橘子花瓶', '创意橘子水瓶', '创意橘子花盆']`

#### 数组核心要点

- 数组索引从0开始（第一个数据索引为0，第二个为1，以此类推）
- 访问数组元素：`数组名[索引]`，如`titles[0]`获取“创意橘子花瓶”

### 4.4 对象

#### 核心目标

一次性存储多个不同类型的数据

#### 对象语法规则

1. 先通过接口约定对象结构和类型：

```TypeScript
interface 接口名 {
  属性1: 类型;
  属性2: 类型;
  // ...更多属性
}
```

2. 声明对象并赋值：

```TypeScript
let 对象名: 接口名 = {
  属性1: 值1,
  属性2: 值2,
  // ...更多属性值
}
```

示例：

```TypeScript
// 约定商品对象结构
interface Goods {
  title: string;
  price: number;
}
// 声明商品对象
let vase: Goods = {
  title: '创意橘子花瓶',
  price: 12.99
}
```

#### 对象核心要点

- 访问对象属性：`对象名.属性名`，如`vase.price`获取12.99
- 区别于数组：数组存同类型数据，对象存不同类型数据

### 4.5 函数

#### 核心目标

封装代码，提升代码复用性

#### 函数语法规则

1. 定义函数：

```TypeScript
function 函数名(参数列表: 类型) {
  // 函数体：要执行的代码
  return 返回值; // 非必须
}
```

1. 调用函数：

```TypeScript
函数名(数据列表); // 传入实际参数
```

示例（计算圆周长）：

```TypeScript
// 定义计算圆周长的函数
function calc(r: number) {
  return 2 * 3.14 * r;
}
// 调用函数
let c1: number = calc(5); // 计算半径为5的圆周长
let c2: number = calc(10); // 计算半径为10的圆周长
```

#### 函数核心要点

- 参数和返回值都不是必须的，根据需求设置
- 一次定义，可多次调用，减少重复代码

### 4.6 箭头函数

#### 核心目标

简化函数写法，提升代码复用性

#### 箭头函数语法规则

```TypeScript
let 函数名 = (参数列表: 类型) => {
  // 函数体
  return 返回值; // 非必须
}
```

示例（计算两数之和）：

```TypeScript
let sum = (num1: number, num2: number) => {
  return num1 + num2;
}
// 调用函数
sum(1, 2); // 结果为3
```

## 五、ArkUI 基础

### 5.1 组件基础

#### 核心目标

掌握组件写法，使用组件布局界面

#### 核心概念

- ArkUI：构建鸿蒙应用界面的框架
- 组件：界面构建与显示的最小单位，分为两种：
  - 容器组件：用于布局（如Column、Row）
  - 内容组件：用于展示内容（如Text）

#### 界面布局思路

**先布局，再内容**（先确定用什么容器排版，再往里面加内容组件）

#### 组件语法规则

- 容器组件：`组件名() { 子组件 }`，如：

  - ```TypeScript
    Column() { // 垂直排列容器
      Text('文本1')
      Text('文本2')
    }
    ```

- 内容组件：`组件名('内容')`，如：`Text('圣诞节氛围感BGM')`

#### 常见组件作用

- Column：内容垂直换行排列
- Row：内容水平一行排列
- Text：展示文本内容

### 5.2 通用属性

#### 核心目标

使用属性美化组件

#### 语法规则

```
组件.属性(值)
```

#### 常用通用属性

| 属性名          | 作用           | 属性值                     |
| --------------- | -------------- | -------------------------- |
| width           | 设置组件宽度   | 数值（默认单位vp）         |
| height          | 设置组件高度   | 数值（默认单位vp）         |
| backgroundColor | 设置组件背景色 | 色值（如'red'、'#FFFFFF'） |

#### 核心要点

- 通用属性所有组件都可使用
- 可链式调用多个属性：`Text('内容').width(200).height(50).backgroundColor('pink')`

### 5.3 文本属性

#### 核心目标

美化文字外观样式

#### 常用文本属性

| 属性名     | 作用         | 属性值                     |
| ---------- | ------------ | -------------------------- |
| fontSize   | 设置字体大小 | 数值（默认单位fp）         |
| fontColor  | 设置文字颜色 | 色值（如'red'、'#FFFFFF'） |
| fontWeight | 设置字体粗细 | 100~900（数值越大越粗）    |

### 5.4 图像组件

#### 核心目标

为界面添加图像资源

#### 图像组件语法

```
Image(图像资源路径)
```

#### 图像资源路径规则

- 本地图片：存放在`resources/base/media/`目录，路径写法：`$r('app.media.图片名')`
- 网络图片：直接写网络链接，如`Image('https://xxx.png')`

### 5.5 内边距与外边距

#### 核心目标

调整组件及内容的位置

#### 核心概念

- 内边距（padding）：拉开内容与组件边缘的距离
- 外边距（margin）：拉开两个组件之间的距离

#### 语法规则

1. 四个方向间距相同：

   1. ```TypeScript
      组件.padding(数值)
      组件.margin(数值)
      ```

2. 四个方向间距不同：

   1. ```TypeScript
      组件.padding({top: 10, bottom: 20, left: 30, right: 40})
      组件.margin({top: 10, bottom: 20, left: 30, right: 40})
      ```

### 5.6 组件边框

#### 核心目标

为组件添加边框效果

#### 边框属性语法

```TypeScript
组件.border({
  width: 粗细, // 如2
  color: 颜色, // 如'red'
  style: 线条样式, // 如实线、虚线
  radius: 圆角 // 如10（数值越大圆角越明显）
})
```

## 六、界面布局实战：歌曲列表

### 布局思路

**先整体，再局部；先布局，再内容，后美化**

### 核心组件与属性

1. 可滚动列表组件：

   1. ```TypeScript
      List() {
        ListItem() { // 列表中的每一项
          // 列表项内容
        }
      }.scrollBar(BarState.Off) // 隐藏滚动条
      ```

2. 尺寸分配属性：`layoutWeight(数字)`

   1. 作用：将外层组件剩余尺寸分成指定份数，当前组件占用对应份数

3. 安全区适配：

   1. ```TypeScript
      组件.expandSafeArea([SafeAreaType.SYSTEM], [SafeAreaEdge.TOP, SafeAreaEdge.BOTTOM])
      ```

   2. 作用：避免内容被手机状态栏、导航栏遮挡
